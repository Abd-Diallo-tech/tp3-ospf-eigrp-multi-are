1. Configurations de base 
NB : j’ai utilisé un mot de passe très faible ici pour le line console et vty mais il est déconseiller de le faire donc dans un vrai réseau il ne faut jamais utiliser ce genre de mot de passe et jamais le divulguer. 

1.1. Passer en mode privilégié  
Router>enable 

1.2. Passer en mode de configuration globale
Router#configure terminal 

1.3. Renommer le routeur
Router(config)#hostname LAB_C

1.4. Attribution de mot passe pour l’accès par console et ssh ou telnet et pour passe en mode privilégie 
LAB_C(config)#line console 0
LAB_C(config-line)#password 1234
LAB_C(config-line)#exit
LAB_C(config)#line vty 0 4
LAB_C(config-line)#password 1234
LAB_C(config-line)#exit
LAB_C(config)#service password-encryption 
LAB_C(config)#enable secret 1234
LAB_C(config)#

1.5. Configuration des interfaces et activation du routeur

LAB_C(config)#interface gigabitEthernet 0/0
LAB_C(config-if)#ip address 192.168.3.253 255.255.255.0
LAB_C(config-if)#no shutdown
LAB_C(config-if)#exit

LAB_C(config)#interface gigabitEthernet 0/1
LAB_C(config-if)#ip address 192.168.4.254 255.255.255.0
LAB_C(config-if)#no shutdown
LAB_C(config-if)#exit
LAB_C(config)#

LAB_C(config)#interface serial 0/1/0
LAB_C(config-if)#ip address 192.168.9.253 255.255.255.0
LAB_C(config-if)#no shutdown
LAB_C(config-if)#exit
LAB_C(config)#



2. Mise en œuvre du routage OSPF
LAB_C(config)#router ospf 1
LAB_C(config-router)#network 192.168.3.0 0.0.0.255 area 0
LAB_C(config-router)#network 192.168.4.0 0.0.0.255 area 0
LAB_C(config-router)#network 192.168.9.0 0.0.0.255 area 0 
LAB_C(config-router)#exit
LAB_C(config)#
2.1. Création d’une interface de bouclage appelée loopback 0
LAB_C(config)#interface loopback 0
LAB_C(config-if)#ip address 3.3.3.3 255.255.255.255
LAB_C(config-if)#exit
LAB_C(config)#router ospf 1
LAB_C(config-router)#rout
LAB_C(config-router)#router-id 3.3.3.3
LAB_C(config-router)#end
LAB_C#clear ip ospf process
Reset ALL OSPF processes? [no]: yes
LAB_C#
