1. Configurations de base 
NB : j’ai utilisé un mot de passe très faible ici pour le line console et vty mais il est déconseiller de le faire donc dans un vrai réseau il ne faut jamais utiliser ce genre de mot de passe et jamais le divulguer. 

1.1. Passer en mode privilégié  
Router>enable 

1.2. Passer en mode de configuration globale
Router#configure terminal 

1.3. Renommer le routeur
Router(config)#hostname LAB_D

1.4. Attribution de mot passe pour l’accès par console et ssh ou telnet et pour passe en mode privilégie 
LAB_D(config)#line console 0
LAB_D(config-line)#password 1234
LAB_D(config-line)#exit
LAB_D(config)#line vty 0 4
LAB_D(config-line)#password 1234
LAB_D(config-line)#exit
LAB_D(config)#service password-encryption 
LAB_D(config)#enable secret 1234
LAB_D(config)#

1.5. Configuration des interfaces et activation du routeur

LAB_D(config)#interface gigabitEthernet 0/0
LAB_D(config-if)#ip address 192.168.5.254 255.255.255.0
LAB_D(config-if)#no shutdown
LAB_D(config-if)#exit
LAB_D(config)#

LAB_D(config)#interface gigabitEthernet 0/1
LAB_D(config-if)#ip address 192.168.4.253 255.255.255.0
LAB_D(config-if)#no shutdown
LAB_D(config-if)#exit
LAB_D(config)#

LAB_D(config)#interface serial 0/1/0
LAB_D(config-if)#ip address 192.168.9.254 255.255.255.0
LAB_D(config-if)#no shutdown
LAB_D(config-if)#exit
LAB_D(config)#


2. Mise en œuvre du routage OSPF
LAB_D(config)#router ospf 1
LAB_D(config-router)#network 192.168.5.0 0.0.0.255 area 0
LAB_D(config-router)#network 192.168.9.0 0.0.0.255 area 0
LAB_D(config-router)#network 192.168.4.0 0.0.0.255 area 0
LAB_D(config-router)#exit
LAB_D(config)#
2.1. Création d’une interface de bouclage appelée loopback 0
LAB_D(config)#interface loopback 0.
LAB_D(config-if)#ip address 4.4.4.4 255.255.255.255
LAB_D(config-if)#exit
LAB_D(config)#router ospf 1
LAB_D(config-router)#router-id 4.4.4.4
LAB_D(config-router)#end
LAB_D#clear ip ospf process
Reset ALL OSPF processes? [no]: yes
LAB_D#

3. Sauvegarder 

LAB_D#
